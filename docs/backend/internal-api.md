# Internal API Communication Framework

The Internal API is the central message backbone of LUMA Smart Home.
Every interaction between engines — commands, events, broadcasts — flows through it.
No engine ever imports or calls another engine directly.

---

## Components

```
┌─────────────────────────────────────────────────┐
│              InternalAPIGateway                 │
│  registerEngine  sendCommand  broadcastMessage  │
│  sendEvent       publishMessage  discoverEngine │
│  getEngineStatus syncMessages  queueOffline…   │
└────────────────────┬────────────────────────────┘
                     │ delegates to
┌────────────────────▼────────────────────────────┐
│                MessageBus                       │
│  (Node.js EventEmitter)                         │
│  publish  subscribe  enqueueOffline             │
│  retryFailedMessages  drainQueueFor             │
│  syncMessages  getDeadLetters                   │
└─────────────────────────────────────────────────┘
```

**Gateway** — high-level API used by engines and REST routes.
**MessageBus** — low-level pub/sub transport. Engines never touch it directly.

---

## Message Format

Every message in the system has this exact shape:

```typescript
interface InternalMessage {
  id:            string;           // UUID, auto-generated by gateway
  source:        EngineId | "gateway";
  destination:   EngineId | "broadcast";
  type:          "COMMAND" | "EVENT" | "QUERY" | "RESPONSE" | "BROADCAST";
  action:        string;           // e.g. "TURN_ON", "FIRMWARE_UPDATED"
  payload:       Record<string, unknown>;
  timestamp:     string;           // ISO 8601
  priority:      "critical" | "high" | "normal" | "low";
  correlationId?: string;          // links request ↔ response
  ttl?:          number;           // ms until message expires in offline queue
}
```

### Message types

| Type | Direction | Use |
|---|---|---|
| `COMMAND` | caller → target | Tell an engine to do something |
| `EVENT` | engine → interested parties | Something happened; engines react |
| `QUERY` | caller → target | Ask for data; expect a RESPONSE back |
| `RESPONSE` | target → caller | Reply to a QUERY (set `correlationId`) |
| `BROADCAST` | engine → all engines | System-wide notification |

### Priority levels

| Priority | Use |
|---|---|
| `critical` | Rollback, emergency shutdown |
| `high` | Commands, connect/disconnect events |
| `normal` | State updates, config changes |
| `low` | Telemetry, stats |

## Gateway API Reference

### `registerEngine(registration, handler): string`

Registers an engine with the gateway. Returns an auth token needed to later unregister.
Called once per engine during `start()`.

```typescript
const token = gateway.registerEngine(
  {
    id: "my_engine",
    name: "My Engine",
    version: "1.0.0",
    capabilities: ["thing_a", "thing_b"],
    subscribedActions: ["DO_THING", "THING_DONE"],
  },
  (message) => this.handleMessage(message),
);
```

---

### `sendCommand(source, destination, action, payload?, priority?): string`

Sends a `COMMAND` message to a specific engine.
Returns the generated message ID.

```typescript
gateway.sendCommand(
  "device_engine",        // who is sending
  "mqtt_engine",          // who should receive
  "PUBLISH_DEVICE_STATE", // what to do
  { topic: "luma/device/lamp1/state", payload: { on: true } },
  "high",
);
```

---

### `sendEvent(source, destination, action, payload?, priority?): string`

Sends an `EVENT` message — something happened that another engine should know about.

```typescript
gateway.sendEvent(
  "firmware_engine",
  "device_engine",
  "FIRMWARE_UPDATED",
  { deviceId: "ESP32_Lamp_01", newVersion: "2.1.0" },
);
```

---

### `broadcastMessage(source, action, payload?, priority?): string`

Sends a `BROADCAST` to every registered engine simultaneously.

```typescript
gateway.broadcastMessage(
  "wifi_engine",
  "WIFI_CONNECTED",
  { ssid: "LUMA_Home_5G", localIP: "192.168.1.42" },
  "high",
);
```

---

### `publishMessage(partial): string`

Low-level publish. Use when you need to set `type`, `correlationId`, or `ttl` manually.

```typescript
gateway.publishMessage({
  source:        "device_engine",
  destination:   "mqtt_engine",
  type:          "QUERY",
  action:        "GET_BROKER_STATUS",
  payload:       {},
  priority:      "normal",
  correlationId: originalMessageId,
});
```

---

### `queueOfflineMessage(source, destination, action, payload?, ttlMs?): string`

Queues a message for delivery when the target engine comes online.
The message expires after `ttlMs` (default 30 s) and moves to the dead-letter queue.

```typescript
gateway.queueOfflineMessage(
  "device_engine",
  "mqtt_engine",
  "PUBLISH_DEVICE_STATE",
  { topic: "luma/device/lamp1/state", payload: { on: false } },
  60_000, // 60 second TTL
);
```

---

### `discoverEngine(engineId): EngineInfo | null`

Returns full info for a registered engine, or null if not found.

```typescript
const info = gateway.discoverEngine("mqtt_engine");
// { id, name, version, status, messagesSent, messagesReceived, ... }
```

---

### `getEngineStatus(engineId): EngineStatus | "unknown"`

Returns `"initializing" | "running" | "stopped" | "error" | "unknown"`.

---

### `getAllEngines(): EngineInfo[]`

Returns all registered engines — useful for health dashboards.

---

### `syncMessages(): InternalMessage[]`

Returns all messages currently waiting in the offline queue.

## Message Bus — Offline Queue & Retry

The message bus handles delivery failures automatically:

```
Message published to unavailable engine
            │
            ▼
       Offline Queue
    (up to 500 messages)
            │
    Every 2 seconds:
            │
            ├── Target still offline → exponential backoff retry
            │       base: 1s, x2 each attempt, up to maxAttempts (5)
            │
            ├── Target now online → deliver immediately
            │
            ├── TTL expired → dead-letter queue (60s retention)
            │
            └── Max retries reached → dead-letter queue
```

Dead-letter messages are visible at `GET /api/engines/queue/dead-letters` and are
purged automatically after 60 seconds.

## BaseEngine — How Engines Are Built

All 6 server-side engines extend `BaseEngine`:

```typescript
import { BaseEngine } from "../base-engine";
import type { EngineId, InternalMessage } from "../../internal-api/types";

export class MyEngine extends BaseEngine {
  readonly id: EngineId = "my_engine";         // must match EngineId type
  readonly name = "My Engine";
  readonly version = "1.0.0";
  readonly capabilities = ["capability_a"];
  readonly subscribedActions = ["DO_THING", "GET_STATUS"];

  // Called once by start()
  protected onStart(): void {
    // initialize state, seed data, connect to external services
  }

  // Called once by stop()
  protected onStop(): void {
    // teardown, close connections
  }

  // Called for every message delivered to this engine
  protected handleMessage(message: InternalMessage): void | Promise<void> {
    switch (message.action) {
      case "DO_THING":
        this.doThing(message);
        break;
      case "GET_STATUS":
        this.replyStatus(message);
        break;
    }
  }

  private doThing(message: InternalMessage): void {
    // do work...
    // notify other engines via helpers from BaseEngine:
    this.send("mqtt_engine", "PUBLISH", { topic: "...", payload: {} });
    this.emit("device_engine", "THING_DONE", { result: "ok" });
    this.broadcast("THING_HAPPENED", { detail: "..." });
  }
}
```

### BaseEngine helper methods

| Method | Maps to gateway method |
|---|---|
| `this.send(dest, action, payload?, priority?)` | `sendCommand` |
| `this.emit(dest, action, payload?, priority?)` | `sendEvent` |
| `this.broadcast(action, payload?, priority?)` | `broadcastMessage` |
| `this.queueOffline(dest, action, payload?, ttl?)` | `queueOfflineMessage` |
| `this.setStatus(status)` | `gateway.setEngineStatus` |

## Engine Lifecycle

```
new MyEngine()
     │
     ▼
engine.start()
  ├── gateway.registerEngine(…, handler)  ← subscribes to messages
  ├── gateway.setEngineStatus("running")
  └── onStart()

     (engine is live and receiving messages)

     │
     ▼
engine.stop()
  ├── onStop()
  ├── gateway.unregisterEngine(id, token)
  └── gateway.setEngineStatus("stopped")
```

Engines are started in `src/engines/index.ts` via `startAllEngines()`.
